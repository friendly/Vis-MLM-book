# Task: Author Index Generation

## Problem summary

The book needs an **Author Index** listing cited authors and the pages where they are cited.
In traditional LaTeX/BibTeX workflows this is handled by the `authorindex` Perl script, which:

1. Reads the `.aux` file produced by LaTeX (which contains `\bibcite` entries from BibTeX)
2. Reads the `.bib` file(s) to get author names
3. Writes a `.ain` file containing `\indexentry{LastName, First}{page}` entries
4. LaTeX then typesets this like a normal index via `\printauthorindex`

---

## Why this is hard with Quarto / pandoc (vs. `.Rnw`)

In my previous book ([*Discrete Data Analysis with R*](https://www.taylorfrancis.com/books/mono/10.1201/b19022/discrete-data-analysis-michael-friendly-david-meyer)), chapters were written in `.Rnw` format.
This was essentially text written in LaTeX (`\chapter{}, `\begin{itemize}, \emph{}, $math$, ...`)
with embedded R code chunks (Sweave/knitr). The compilation pipeline was:

```
.Rnw ŌåÆ knitr ŌåÆ .tex ŌåÆ LaTeX ŌåÆ BibTeX ŌåÆ LaTeX ŌåÆ authorindex ŌåÆ LaTeX
```

Every step was explicit and controllable:
- `\cite{key}` commands appeared directly in the `.tex` source
- BibTeX was called as a normal step, writing `\bibdata{...}` and `\bibcite{...}` to the `.aux` file
- The `authorindex` package intercepted `\cite` calls (via its `\aicite` wrapper, or by the author
  consistently using `\aicite` instead of `\cite`)
- The Perl script could find everything it needed in the `.aux` file

With Quarto / pandoc the pipeline is fundamentally different:

```
.qmd ŌåÆ knitr ŌåÆ .md ŌåÆ pandoc (citeproc) ŌåÆ .tex ŌåÆ LaTeX (no BibTeX step) ŌåÆ ŌĆ”
```

Pandoc has its own built-in citation processor (**citeproc**) that resolves all citations
*before* LaTeX ever runs. This means:

1. **No `\cite{}` commands in the `.tex` output** ŌĆö instead, pandoc emits
   `\citeproc{ref-key}{Formatted Author, Year}`, a custom macro that produces the formatted
   reference text directly. Standard `\cite{}` is called internally, but only as a no-op
   for cross-referencing purposes.

2. **No BibTeX step** ŌĆö pandoc reads `.bib` files itself and produces a
   `\begin{thebibliography}...\end{thebibliography}` block in the `.tex` file. LaTeX never
   calls BibTeX, so `\bibdata{...}` is never written to the `.aux` file.

3. **Keys are renamed** ŌĆö pandoc prefixes all citation keys with `ref-`, so a BibTeX entry
   with key `Friendly2013` becomes `ref-Friendly2013` in the `.tex` and `.aux` files.
   The `.bib` files still use the original keys.

4. **`authorindex.sty` doesn't auto-patch `\cite`** ŌĆö despite what its documentation implies
   for typical use, the package only provides `\aicite` as an alternative to `\cite`. It does
   *not* globally redefine `\cite` to write `\citationpage` entries. In a `.Rnw` workflow the
   author would use `\aicite` throughout (or the package was used with natbib which provides
   hooks); in Quarto, there is no opportunity to use `\aicite` since citation syntax is
   `[@key]` and the `.tex` is generated by pandoc.

5. **Windows CRLF line endings** ŌĆö LaTeX on Windows writes `.aux` files with CRLF line endings.
   The `authorindex` Perl script was written for Unix and its regexes use `$` to match
   end-of-line; on Windows, every line ends with `\r\n`, so `\r` sits before `\n` and the
   `$` anchor fails to match, breaking all pattern matching silently.

The net result: every piece of the `authorindex` workflow was broken by the Quarto/pandoc
pipeline, and the failures compounded in non-obvious ways.

## Approaches / Solutions

### Option A: Fix the Perl script invocation (minimal change) ŌåÉ **SOLVED**

#### Changes made (2026-04-04 / 2026-04-05)

**`latex/preamble.tex`:**

1. Removed the duplicate `\usepackage[small,firstabbrev]{authorindex}` (was at both line 136
   and line 176). Kept the first instance.

2. Added an `\AtBeginDocument` hook immediately after `\aimaxauthors{5}` that:
   - Patches `\@citex` (LaTeX's internal cite dispatcher) to also call `\@aicitey`, which
     writes `\citationpage{key}{page}` entries to the `.aux` file. This is necessary because
     pandoc's `\citeproc{key}{text}` calls `\cite{key}` internally, and `authorindex` only
     intercepts `\aicite`, not `\cite`.
   - Writes a `\bibdata{...}` entry to the `.aux` file listing all `.bib` files, so the Perl
     script can tell BibTeX which databases to use (this entry is never written by citeproc
     since BibTeX is not called).

   ```latex
   \AtBeginDocument{%
     \let\@ai@orig@citex\@citex
     \def\@citex[#1]#2{\@ai@orig@citex[#1]{#2}\@aicitey{#2}}%
     \if@filesw\immediate\write\@auxout{%
       \string\bibdata{references,R-refs,pkgs,packages,Rpackages-4.5.1}}\fi}
   ```

**`latex/authorindex_debug`** (working copy; fixes later promoted to `latex/authorindex`):

1. Added `s/\r//g;` at the top of the `while(<>)` input loop to strip Windows CRLF
   line endings before any regex matching.

2. In the `\citationpage` handler: extract key and page into named variables, then
   strip the pandoc `ref-` prefix from the key before all downstream processing.

3. In the `\bibcite` handler: same `ref-` prefix stripping for consistency.

4. Changed the BibTeX exit-code check from a hard `die` to a warning when the `.bbl`
   file was produced anyway ŌĆö "Repeated entry" errors are expected when multiple `.bib`
   files share package keys, and BibTeX still produces usable output.

**`make-authorindex.sh`** (project root):

Shell script that sets `BIBINPUTS` and `BSTINPUTS` and runs the script:
```bash
export BIBINPUTS="C:/R/Projects/Vis-MLM-book/bib"
export BSTINPUTS="C:/Users/friendly/AppData/Roaming/TinyTeX/texmf-dist/bibtex/bst"
perl latex/authorindex -d index
```
*(Note: script was updated 2026-04-05 to call `latex/authorindex` after fixes were
promoted from `authorindex_debug`. The debug copy is retained for further debugging.)*

**`latex/after-body.tex`:**

Added `\renewenvironment{theauthorindex}` before `\printauthorindex` to enforce
single-line spacing and remove inter-entry paragraph space (the default environment
produced double-spaced output). Per authorindex docs, redefining this environment
is the recommended way to control index formatting.

**`latex/preamble.tex`** (2026-04-05):

3. Added `\providecommand{\de}[1]{d'#1}` near the `authorindex` setup. This was
   previously defined via `@Preamble` in `references.bib`, which worked in `.Rnw`
   because BibTeX emitted preamble content into the `.bbl` file read by LaTeX. With
   pandoc citeproc, `.bib` files are read by pandoc and `@Preamble` directives are
   silently discarded ŌĆö they never reach LaTeX. Author names using `\de{}` (e.g.
   `{\de{O}}cagne`) caused "Undefined control sequence" errors during author index
   typesetting. Fix: define the command directly in `preamble.tex`.

### NEW PROBLEM (2026-04-07): `\@citex` patch breaks PDF build ŌĆö deadlock

#### Symptom

```
! Extra }, or forgotten \endgroup.
l.4211 ...citeproc{ref-Blishen-etal-1987}{1987}).}
```

LaTeX stops at page 74, writing an incomplete `.aux` file. Next build fails for
the same reason (citation still undefined). **Deadlock.**

#### Root cause

The `\@citex` patch (Option A above) calls `\@aicitey{key}` inside the same
invocation as `\@ai@orig@citex`. When `\citeproc{key}{text}` is called inside
`\footnote{}`, the call sequence is:

```
\footnote{ ... \citeproc{key}{text} ... }
    Ōå│ \begingroup                          ŌåÉ from \citeproc
        \cite{key}
            Ōå│ \@citex[]{key}              ŌåÉ patched version calls \@aicitey{key}
                  group imbalance here when citation is undefined on first pass
    \endgroup
}                                          ŌåÉ LaTeX says "Extra }"
```

The group balance breaks when `\@citex` processes an undefined citation inside
the `\begingroup...\endgroup` + `\footnote{}` combination. The error is fatal:
LaTeX stops after page 74, writing an incomplete `.aux`. Since `\bibcite`
entries are only written at the end of the document (where the bibliography is),
the next build also has undefined citations ŌåÆ same error ŌåÆ deadlock.

**Why Tufte:83 footnote (ch. 3, line 2065) did NOT trigger the error:**
Its `\bibcite` entry survived in the `.aux` from an even-earlier successful
build. The Blishen-etal-1987 entry was lost when the `.aux` was truncated by
the first failed build.

**Why citations in footnotes "used to work":**
Before the `\@citex` patch was added (2026-04-05), `\citeproc` in footnotes
worked fine. The patch is what broke them. The first build after adding the
patch went far enough to truncate the `.aux`, and subsequent builds were
trapped in the deadlock.

#### Fix applied (2026-04-07)

Two changes to `latex/preamble.tex`:

1. **Disabled the `\@citex` patch** (commented out the `\let` and `\def` lines
   in `\AtBeginDocument`). This means `\@aicitey` is no longer called on every
   citation, so the author index cannot be regenerated. The pre-built `index.ain`
   is still used for typesetting.

2. **Redefined `\citeproc`** to handle undefined citations gracefully:

   ```latex
   \makeatletter
   \renewcommand\citeproc[2]{%
     \@ifundefined{b@#1}{#2}%
       {\begingroup\def\citeproctext{#2}\cite{#1}\endgroup}%
   }
   \makeatother
   ```

   If the citation is **undefined** (first pass, stale `.aux`): output the
   pre-formatted text `#2` directly ŌĆö no `\begingroup...\endgroup`, no group
   imbalance. If **defined**: use the original mechanism (preserves hyperlinks).

   Pandoc provides the correct formatted text in `#2` regardless, so the output
   is identical either way. This breaks the deadlock permanently.

   The same redefinition was also applied directly to `Vis-MLM.tex` for
   immediate use (Quarto will regenerate Vis-MLM.tex from preamble.tex on the
   next build).

#### Status (2026-04-07): Build fixed; author index disabled

- The `\@citex` patch is **disabled** ŌĆö author index cannot be regenerated
  (would cause the same "Extra }" deadlock).
- The pre-built `index.ain` (from 2026-04-05) is still used for typesetting.
  It will become stale as chapters change.
- To regenerate the author index: re-enable the `\@citex` patch, do a
  **TeXStudio** build (not Quarto ŌĆö Quarto can't resume after first-pass errors),
  run `bash make-authorindex.sh`, then **disable the patch again** before the
  next Quarto build.

#### Alternative fix if author index needs updating frequently

Instead of patching `\@citex`, patch `\citeproc` directly ŌĆö call `\@aicitey`
**before** `\begingroup`, avoiding the group interaction:

```latex
\AtBeginDocument{%
  \let\@ai@orig@citeproc\citeproc
  \RenewDocumentCommand\citeproc{mm}{%
    \@aicitey{#1}%
    \@ai@orig@citeproc{#1}{#2}%
  }%
}
```

This hasn't been tested yet. The risk is that `\@aicitey{key}` expects keys in
`ref-...` Pandoc format; needs verification against the Perl script's key-stripping logic.

### Option B: Rewrite in R (no Perl dependency)

Parse the `.aux` and `.bib` files in R to produce the `.ain` file directly.
The logic is:

1. Read `index.aux`, extract all `\citationpage{key}{page}` entries (after PDF is compiled
   with the Option A `\@citex` patch in place, these will exist)
2. Read the `.bib` files with `bib2df::bib2df()` or `RefManageR` to get author lists per key
3. For each citation key on each page, emit author index entries
4. Write an `.ain` file in the format `authorindex` produces

**Pros:** No Perl, no env var issues, works identically on all machines, easy to debug.
**Cons:** Some effort to write; need to match output format of `authorindex` exactly.
Implement this if Option A remains fragile.

---

### Option C: Use `biblatex` + `biber` instead of BibTeX

`biblatex` has native support for author indexing via `\indexnames` ŌĆö no separate script needed.
However, switching from BibTeX to biblatex/biber is a significant change to the build pipeline
and may conflict with the CRC `krantz` class requirements.

**Pros:** Clean, modern, no external script.
**Cons:** Large migration effort; uncertain compatibility with krantz class and Quarto's PDF pipeline.

# Task: Table of Contents / PDF Bookmark Problems

## Problem summary

Two related issues with the PDF output's table of contents and bookmarks:

1. **Part headings appear in wrong TOC position** â€” In the rendered PDF, Part headings
   visually appear *after* the last chapter of each part rather than *before* the first
   chapter. (Priority: high â€” affects print layout.)

2. **Duplicate PDF bookmarks** â€” The PDF bookmark panel shows multiple entries for
   "Subject Index" and "Author Index". (Priority: low â€” cosmetic.)

---

## Problem 1: Part headings in wrong TOC position

### What we know

The generated `Vis-MLM.tex` has the correct source order:

```latex
\part{Orienting Ideas}          % line 1086
\chapter{Warm-up Exercises}     % line 1088
...
\part{Exploratory Methods}      % line 3079
\chapter{Plots of Multivariate Data}  % line 3081
```

The generated `index.toc` also has the correct order:

```
\contentsline {part}{I\hspace {1em}Orienting Ideas}{1}{part.1}%
\contentsline {chapter}{\numberline {1}Warm-up Exercises}{3}{chapter.1}%
...
\contentsline {part}{II\hspace {1em}Exploratory Methods}{39}{part.2}%
\contentsline {chapter}{\numberline {4}Plots of Multivariate Data}{41}{chapter.4}%
```

So the problem is in how `krantz` **renders** `\contentsline{part}` entries from the
`.toc` file â€” not in the source or `.toc` ordering.

### Root cause (confirmed 2026-04-05)

**This is intentional `krantz` class behaviour, not a Quarto bug.**

Inspecting `krantz.cls` (project root), lines 1017â€“1021:

```latex
\let\toc@draw\relax        % initially a no-op

\def\l@part#1#2{%
  \toc@draw                % execute pending part (nothing on first call)
  \gdef\toc@draw{\draw@part{\large #1}{\large #2}}}  % store THIS part
```

Each `\l@part` call stores the current part's drawing command in `\toc@draw` but does
**not** execute it. `\toc@draw` only fires at the *start* of the **next** `\l@part` call.
Because `\l@chapter` never calls `\toc@draw`, the Part I heading is drawn only when
Part II is encountered â€” after Chapters 1, 2, 3 have already been typeset in the TOC.
The last part's heading (`\toc@draw` after Part V) is never drawn at all, since
there is no subsequent `\l@part` to trigger it.

In contrast, the standard `book` class `\l@part` draws immediately with no deferral.
This is purely a `krantz` design choice.

### Proposed fix

Patch `\l@chapter` in `preamble.tex` to flush `\toc@draw` before drawing each chapter.
This makes each Part heading appear immediately before the first chapter in its part:

```latex
\makeatletter
\let\@krantz@l@chapter\l@chapter
\renewcommand*\l@chapter[2]{%
  \toc@draw\let\toc@draw\relax   % flush any pending part heading
  \@krantz@l@chapter{#1}{#2}}
\makeatother
```

The `\let\toc@draw\relax` after calling it prevents double-execution if two chapters
follow without an intervening part.

### Things to try

- [ ] Apply the `\l@chapter` patch above in `preamble.tex` and recompile.
- [ ] Check that the last Part (Appendices) also gets flushed â€” it would be triggered
  by the Subject Index `\l@chapter` entry, which is the first chapter-level entry
  after the Appendices `\l@part`.
- [ ] Verify with CRC that changing the TOC layout is acceptable for the house style,
  or ask if they have an updated `krantz.cls` that handles this differently.

## Problem 2: Duplicate PDF bookmarks for indexes

### What we know

The PDF bookmark panel shows multiple entries for "Subject Index" and "Author Index".

**Subject Index** is generated by `\printindex` in `after-body.tex` (line 2). The
`makeindex` / `imakeidx` package calls `\chapter*{Index}` internally, which hyperref
automatically bookmarks. If `\printindex` also calls `\addcontentsline` internally,
that can generate a second bookmark.

**Author Index** is generated by these lines in `after-body.tex`:
```latex
\addcontentsline{toc}{chapter}{Author Index}
\chapter*{Author Index}
```
With hyperref active, `\chapter*{Author Index}` creates one PDF bookmark, and
`\addcontentsline{toc}{chapter}{...}` can create another. The standard fix is to use
`\phantomsection` before `\addcontentsline` and put `\addcontentsline` *after*
`\chapter*`, or to explicitly suppress one of the bookmark sources.

### Things to try

- [ ] Reorder `after-body.tex` to put `\addcontentsline` after `\chapter*`:
  ```latex
  \chapter*{Author Index}
  \addcontentsline{toc}{chapter}{Author Index}
  ```
- [ ] If still duplicating, add `\phantomsection` before `\addcontentsline`:
  ```latex
  \phantomsection
  \addcontentsline{toc}{chapter}{Author Index}
  \chapter*{Author Index}
  ```
- [ ] For Subject Index: check if `imakeidx` option `intoc` is set; if so, disable it and
  add an explicit `\addcontentsline` instead to avoid the duplicate.
- [ ] Check `preamble.tex` for `\usepackage{imakeidx}` options â€” `intoc=true` combined
  with an explicit `\addcontentsline` in `after-body.tex` would produce duplicates.
